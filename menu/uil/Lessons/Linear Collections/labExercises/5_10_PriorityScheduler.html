6.10 PriorityScheduler

Description

Your assignment is to write a Java program that simulates a computer operating system performing event scheduling. When an application makes a request to the operating system, the requested task or job is assigned a priority level (some tasks are more urgent than others) and placed in a scheduling queue to wait its turn to be processed. The job with the highest priority is executed first, then the job with the second highest priority is executed second, and so on. If a new job is added to the event queue then it will executed according to its priority level.

Your program should do the following:

  • Step 1: Create priority queue (job queue)
    • Create a priority queue and add 10 jobs. In the simulation a job is represented by a random number in the range of 1 to 100. The smaller the number the higher the priority.
  • Step 2: Run scheduling cycle
    • The program simulates the scheduling cycle of the operating system by dispatching the highest priority jobs while the random number generator continues generating numbers and creating new jobs.
    • Each time through the scheduling cycle the program should print the dispatched job, the newly created job, and the jobs still remaining in the queue.
  • Step 3: Empty queue
    • The scheduling cycle should stop after 20 cycles and then dispatch the remaining jobs still in the queue.

class java.util.PriorityQueue<E>

boolean add(E obj) adds obj to the priority queue; returns true
E remove() removes and returns the minimal element from the priority queue
E peek() returns the minimal element from the priority queue
boolean isEmpty()
